## 0) Data and metadata

- Train/official test: `BreastCA-img/{train,test}/metadata_laplacian.json` and similar files are **JSONL** (one JSON per line).
- Extrapolation for three public sets: `breastca_png_aligned` reads **JSONL only**. Requires
  `BreastCA-img/infer_public_<NAME>/test/metadata_laplacian.json`. If missing, convert `infer/<NAME>/metadata_<NAME>_laplacian.json` (JSON array) to JSONL:

  cd /n/holylfs05/LABS/zhuang_lab/Lab/Jiaming/US2SWEdiff/BreastCA-img
  for NAME in BLUSG BUSBRA BUSI; do
    mkdir -p "infer_public_${NAME}/test"
    python3 -c "import json; n='${NAME}'; s=json.load(open(f'infer/{n}/metadata_{n}_laplacian.json')); o=open(f'infer_public_{n}/test/metadata_laplacian.json','w'); [o.write(json.dumps(r)+'\n') for r in s]"
  done

## 1) Train (Laplacian->SWE, 256, example)

Run in the `BBDM-MICCAI` root (step count and save interval can be adjusted as needed; consistent with `configs/BBDM_png_base.yaml`):

  source "$(conda info --base)/etc/profile.d/conda.sh"
  conda activate controlnet_new
  cd /n/holylfs05/LABS/zhuang_lab/Lab/Jiaming/US2SWEdiff/BBDM-MICCAI

  python main.py \
    -c configs/BBDM_png_base.yaml \
    -r results \
    --train \
    --exp_name png_laplacian2swe_true256_b5 \
    --gpu_ids 0 \
    --HW 256 \
    --batch 15 \
    --metadata_name metadata_laplacian.json \
    --source_modality laplacian \
    --valid_split test \
    --test_split test \
    --max_steps 4000 \
    --save_step_interval 1000

Weights and log root directory (relative to `BBDM-MICCAI`):

  /n/holylfs05/LABS/zhuang_lab/Lab/Jiaming/US2SWEdiff/BBDM-MICCAI/results/breastca_png_256/png_laplacian2swe_true256_b5/

(To resume from midway, add `--resume_model` / `--resume_optim` pointing to files under `checkpoint/` in this directory.)
